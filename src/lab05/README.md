# Лабораторная работа №5 — Функции как аргументы. Стратегии и делегаты.

## Цель работы

Лабораторная работа посвящена передаче функций как аргументов, функциям высшего порядка (map, filter, sorted), паттерну «Стратегия», lambda-выражениям и интеграции функционального стиля с объектно-ориентированным кодом из предыдущих лабораторных.

## Реализованные функции и стратегии

**Функции-стратегии сортировки**

Реализовано несколько стратегий сортировки: by_title — сортировка по названию книги, by_author — сортировка по автору, by_year — сортировка по году издания, by_price — сортировка по цене, а также by_price_then_title — сортировка сначала по цене, затем по названию.

**Функции-фильтры**

Для фильтрации коллекции реализованы функции: is_available — оставляет только доступные книги, is_borrowed — только выданные книги, is_printed — только печатные книги, is_ebook — только электронные книги, is_audiobook — только аудиокниги.

**Функции высшего порядка и фабрики**

Созданы фабрики функций: make_price_filter(max_price) — создаёт фильтр по максимальной цене, make_year_filter(min_year, max_year) — создаёт фильтр по диапазону годов, make_discount_strategy(percent) — создаёт стратегию применения скидки.

**Паттерн «Стратегия» и callable-объекты**

В лабораторной работе реализован паттерн «Стратегия» через callable-объекты. Стратегия — это взаимозаменяемый алгоритм, который можно передать в коллекцию без изменения её кода. Созданы следующие callable-объекты: DiscountStrategy(percent) — стратегия применения скидки, при вызове применяет скидку к книге; PriceExtractor() — стратегия извлечения цены; TitleExtractor() — стратегия извлечения названия.

Преимущества паттерна «Стратегия»: стратегии взаимозаменяемы без изменения кода коллекции, легко добавлять новые стратегии, код становится более гибким и тестируемым.

## Вывод

В ходе лабораторной работы изучены:

- **Передача функций как аргументов** — методы sort_by(), filter_by(), apply() коллекции принимают функции-стратегии
- **lambda, map, filter** — lambda для простых функций, map для преобразования коллекции, filter для фильтрации
- **Функции высшего порядка и замыкания** — фабрики функций (make_price_filter, make_year_filter) возвращают функции с захваченными параметрами
- **Паттерн «Стратегия»** — реализован через callable-объект DiscountStrategy, стратегии взаимозаменяемы без изменения кода коллекции
