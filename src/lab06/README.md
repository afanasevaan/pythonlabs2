# Лабораторная работа №6 — Generics и typing

## 1. Цель работы

Освоить систему аннотаций типов в Python (typing). Научиться создавать обобщённые (generic) классы с помощью TypeVar и Generic. Понять концепцию структурной типизации через typing.Protocol.

## 2. Описание реализованных типов и контейнеров

**Generic-класс TypedCollection**

Это обобщённая версия коллекции из ЛР-2. Она параметризована типом T (TypeVar), что позволяет хранить элементы любого типа, но с гарантией единообразия. Коллекция поддерживает методы add, remove, find, filter, map, sort_by, filter_by, apply.

**TypeVar**

- `T = TypeVar('T')` — базовый тип элементов коллекции
- `R = TypeVar('R')` — тип результата преобразования в методе map
- `D = TypeVar('D', bound=Displayable)` — ограничение: только объекты с методом display()
- `S = TypeVar('S', bound=Scorable)` — ограничение: только объекты с методом score()

**Protocols (структурная типизация)**

Созданы два протокола: Displayable (требует метод display() -> str) и Scorable (требует метод score() -> float). Классы PrintedBook, Ebook, AudioBook не наследуют эти протоколы явно, но имеют соответствующие методы, поэтому автоматически им соответствуют. Это демонстрирует структурную типизацию: важна не иерархия наследования, а наличие нужных методов.

## 4. Вывод

В ходе лабораторной работы изучены:

- **Аннотации типов** — параметры конструктора, возвращаемые значения, атрибуты. Типизация делает код самодокументируемым и помогает IDE подсказывать методы.

- **Generic и TypeVar** — создание обобщённых коллекций, которые работают с любым типом данных, обеспечивая при этом безопасность типов.

- **Protocol (структурная типизация)** — объект подходит под протокол, если у него есть нужные методы, даже без явного наследования. Это делает код более гибким.

- **Как типизация помогает** — читаемость (сразу видно, какие типы ожидаются), поддержка (IDE подсвечивает ошибки), документация (аннотации заменяют комментарии).
