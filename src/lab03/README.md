# Лабораторная работа №3: Наследование и иерархия классов

## 1. Цель работы

- Освоить механизм наследования классов
- Научиться строить иерархию объектов
- Понять разницу между базовым и производным классами
- Научиться переиспользовать код
- Освоить переопределение методов

## 2. Описание реализованной иерархии классов

### Иерархия

```text
Book
├── PrintedBook (печатная книга)
├── Ebook (электронная книга)
└── AudioBook (аудиокнига)
```
## 3 Описание иерархии классов

**Базовый класс (Book)** — из ЛР-1: title, author, year, pages, price, is_available. Методы: borrow(), return_book(), get_age_category().

**Дочерние классы:**

| Класс | Новые атрибуты | Новый метод |
|-------|----------------|-------------|
| PrintedBook | binding (переплёт), circulation (тираж) | get_shipping_weight() |
| Ebook | file_format (формат), file_size_mb (размер) | get_download_time() |
| AudioBook | duration_hours (длит-ть), narrator (чтец) | get_remaining_time() |

**Различия:** 
Базовый класс — это "книга вообще" (абстракция).  
Дочерний класс — это "конкретная книга с конкретными особенностями" (реализация).

Дочерний класс расширяет базовый: добавляет то, чего у родителя нет, но при этом имеет всё, что есть у родителя.

## Сценарий 1: Создание объектов разных типов

Созданы три объекта:
- **PrintedBook** — "Война и мир" (твердый переплёт, тираж 5000)
- **Ebook** — "1984" (формат EPUB, 2.5 МБ)
- **AudioBook** — "Мастер и Маргарита" (читает Хабенский, 12.5 ч)

Каждый объект выводится в своём формате благодаря переопределённому методу `__str__`.

## Сценарий 2: Коллекция + полиморфизм

В коллекцию добавлены три объекта:

- **PrintedBook** — "Война и мир" (вывод в формате печатной книги)

- **Ebook** — "1984" (вывод в формате электронной книги)

- **AudioBook** — "Мастер и Маргарита" (вывод в формате аудиокниги)

Метод `isinstance()` правильно определил тип каждого объекта.

## Сценарий 3: Уникальные методы

Вызваны методы, специфичные для каждого дочернего класса:

- **PrintedBook** — `get_shipping_weight()`: вес книги "Война и мир" составил 13.00 кг (1300 страниц × 0.01 кг)

- **Ebook** — `get_download_time()`: время скачивания "1984" — 2.00 секунды (при скорости 10 Мбит/с)

- **AudioBook** — `get_remaining_time()`: осталось слушать "Мастер и Маргарита" — 9.0 часов (из 12.5 прослушано 3.5)

## 4. Вывод

В ходе выполнения лабораторной работы были изучены:

**Наследование**
- Создание дочерних классов PrintedBook, Ebook, AudioBook на основе базового класса Book
- Использование `super()` для вызова конструктора родительского класса
- Добавление новых атрибутов (binding, circulation, file_format, file_size_mb, duration_hours, narrator)
- Добавление новых методов (get_shipping_weight, get_download_time, get_remaining_time)
- Переиспользование кода базового класса без дублирования

**Полиморфизм**
- Переопределение метода `__str__` для каждого дочернего класса
- Хранение объектов разных типов в одной коллекции Library
- Корректная работа `isinstance()` для определения типа объектов
- Единообразный вызов методов при разных типах объектов
