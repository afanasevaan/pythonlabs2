## 2. Описание интерфейсов. 

В ходе лабораторной работы создано 3 интерфейса (абстрактных класса):

**1. Printable**

Назначение: для объектов, которые можно вывести в виде строки.

Требуемый метод: `to_string() -> str`

Метод должен возвращать строковое представление объекта.

**2. Comparable**

Назначение: для объектов, которые можно сравнивать между собой.

Требуемый метод: `compare_to(other) -> int`

Метод должен возвращать отрицательное число, если текущий объект меньше другого, ноль — если объекты равны, положительное число — если текущий объект больше другого.

**3. Discountable**

Назначение: для объектов, на которые можно применить скидку.

Требуемый метод: `apply_discount(percent: float) -> None`

Метод должен уменьшать цену объекта на указанный процент.

## 3. Реализация в классах

### Какие классы реализуют интерфейсы

Интерфейсы реализуют следующие классы: `Book`, `PrintedBook`, `Ebook`, `AudioBook`.

Все классы реализуют все три интерфейса: Printable, Comparable, Discountable.

### Как отличается поведение

**Printable (метод to_string()):**

- `Book` — "Книга: Название - Автор (Год) | Цена"
- `PrintedBook` — "[Печатная] Название - Автор (Год) | Страницы | Цена | Переплёт"
- `Ebook` — "[Электронная] Название - Автор (Год) | Цена | Формат"
- `AudioBook` — "[Аудио] Название - Автор (Год) | Цена | Читает: Чтец"

**Comparable (метод compare_to()):**

- `Book` — сравнение по цене
- `PrintedBook` — сравнение по году издания
- `AudioBook` — сравнение по длительности

**Discountable (метод apply_discount()):**

- `Book` — скидка от 0 до 100%
- `Ebook` — скидка не более 50%
- `PrintedBook` и `AudioBook` — наследуют поведение от Book
