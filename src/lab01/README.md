# Лабораторная работа №1: Класс и инкапсуляция

## Вариант: Библиотека / Книги

## Описание проекта

Реализация класса `Book` (Книга) для библиотечной системы. 
Класс демонстрирует принципы ООП: инкапсуляцию, свойства, магические методы и бизнес-логику.

## Структура класса

### Атрибуты (закрытые поля)
- `_title` - название книги (строка)
- `_author` - автор книги (строка)  
- `_year` - год издания (целое число)
- `_pages` - количество страниц (целое число)
- `_price` - стоимость книги (число с плавающей точкой)

### Атрибуты класса
- `MAX_PRICE = 10000` - максимальная допустимая цена
- `MIN_YEAR = 1400` - минимальный год издания

### Состояние объекта
- `is_available` - доступность книги (True/False)

### Методы

#### Магические методы
- `__init__` - конструктор с валидацией данных
- `__str__` - строковое представление для пользователя
- `__repr__` - техническое представление для разработчика
- `__eq__` - сравнение книг по названию и автору

#### Свойства (property)
- `title` - геттер и сеттер с валидацией
- `author` - геттер и сеттер с валидацией
- `year` - геттер и сеттер с валидацией
- `pages` - геттер и сеттер с валидацией
- `price` - геттер и сеттер с валидацией

#### Методы валидации
- `validate_title` - проверка названия
- `validate_author` - проверка автора
- `validate_year` - проверка года
- `validate_pages` - проверка страниц
- `validate_price` - проверка цены

#### Бизнес-методы
- `get_age_category()` - определение возрастной категории книги
- `apply_discount(percent)` - применение скидки
- `is_expensive()` - проверка на дорогую книгу
- `borrow()` - выдача книги (изменение состояния)
- `return_book()` - возврат книги (изменение состояния)

## Ответы на вопррсы 
#### Вопрос 1. Что является сущностью?
В коде: Сущностью является Книга (класс Book)
Книга реально существует в библиотеке (предметная область)
У неё есть свои характеристики (название, автор, год...)
Она может что-то делать (её можно выдать, вернуть, применить скидку)
Она может находиться в разных состояниях (доступна/выдана)

#### Вопрос 2. Какие у него атрибуты?
Атрибуты-характеристики (отвечают на вопрос "какой?"):
- `self._title`        # название книги
- `self._author`       # автор
- `self._year`         # год издания
- `self._pages`        # количество страниц
- `self._price`        # цена
- `self.is_available`  # доступность (True/False)

Атрибуты-действия (отвечают на вопрос "что делает?"):
- `def borrow(self)`           # "взять книгу" - действие
- `def return_book(self)`      # "вернуть книгу" - действие
- `def apply_discount(self)`   # "применить скидку" - действие
- `def get_age_category(self)` # "определить возраст" - действие
- `def is_expensive(self)`     # "проверить, дорогая ли" - действие

#### Вопрос 3. Какие инварианты?
- Название книги не пустое
- Имя автора не пустое
- Год выпуска от 1400 до 2026
- Количество страниц > 0
- Цена ≥ 0
- Цена ≤ MAX_PRICE (10000)

#### Вопрос 4. Что значит "равенство"?
- Две книги равны, если у них одинаковое название И одинаковый автор
- Год издания, количество страниц, цена - НЕ важны для равенства

#### Вопрос 5. Есть ли состояние?
Да, есть:
- Книда доступна, она есть в наличии
- книга выдана, она находится у читателя 


Правила:
- Нельзя выдать уже выданную книгу
- При возврате книга снова станосится доступной
