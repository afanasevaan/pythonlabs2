# Лабораторная работа №2: Коллекция объектов

## Цель работы
- Научиться работать с коллекциями объектов
- Понять разницу между моделью сущности и контейнером объектов
- Реализовать собственный контейнерный класс
- Освоить итерацию по объектам
- Реализовать базовые операции управления коллекцией

## Предметная область

**Библиотека / Книги**

Выбранная сущность: **`Book`** → контейнер: **`Library`**


## Защита коллекции
- Внутреннее хранилище `self._items` является защищённым (с одним подчёркиванием)
- Прямой доступ к внутреннему списку извне запрещён
- Метод `get_all()` возвращает копию списка, а не оригинал, предотвращая его модификацию извне
- Метод `add()` проверяет тип добавляемого объекта и не позволяет добавить дубликаты
- Метод `remove()` проверяет наличие объекта в коллекции перед удалением
- Методы `__getitem__` и `remove_at` проверяют индекс на выход за границы

## Реализованный функционал

### Оценка 3
- `add(item)` — добавление объекта с проверкой типа
- `remove(item)` — удаление объекта
- `get_all()` — возврат копии списка всех объектов

### Оценка 4
- `find_by_title()`, `find_by_author()`, `find_by_year()` — поиск по атрибутам
- `__len__()` — поддержка `len()`
- `__iter__()` — поддержка итерации `for item in collection`
- Защита от добавления дубликатов (по названию и автору)

### Оценка 5
- `__getitem__()` — поддержка индексации `collection[index]`
- `remove_at(index)` — удаление по индексу с проверкой границ
- `sort(key, reverse)` — сортировка коллекции
- `filter(predicate)` — фильтрация с возвратом новой коллекции
- `get_available()`, `get_borrowed()`, `get_expensive()`, `get_by_age_category()` — логические операции над коллекцией
